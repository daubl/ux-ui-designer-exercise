Here is a list of the usability issues that have been resolved in version B:
TABS
  • The active tab isn't visible enough. If someone has poor eyesight or a bad screen, the difference between the active tab and a normal tab isn't very obvious.
  • For the tab whose content is a loader, it's a shame that you have to click on it to see it, when all you have to do is add a loader to the tab to realise that the tab's content is not yet available.
  • For the disable tab, it's always good to indicate why an element isn't accessible (by adding a tooltip for example) - Not yet implemented
TITLES
The difference in hierarchy between the 2 titles is not obvious, so I've emphasised it in my version.
CONTENT
It's not clear which item in the list the image is associated with.
UI
I preferred to remove the background colours of the header and footer because it gave too much of an impression that the content was crushed. I've added small image icons for the items in the list, thinking of other use cases like, for example, if one of the images had been a video it would have been indicated by a play icon.
